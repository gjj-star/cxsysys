## Git 自动版本号管理 (00:00~00:12)

### 需求背景
用户反馈每次打包 APK 都显示 1.0，需要动态版本号管理。

### 实现方案：基于 Git Tag 自动生成
| 文件 | 操作 | 说明 |
|------|------|------|
| `version.gradle.kts` | **新建** | 根级版本管理脚本，读取 git tag + commit 数 |
| `build.gradle.kts` (根) | 修改 | 引入 version.gradle.kts |
| `app/build.gradle.kts` | 修改 | defaultConfig 从 extra 读取自动版本号 |
| `MineScreen.kt` | 修改 | 底部显示「版本 x.x.x (Build xxx)」 |
| `README.md` | 重写 | 新增完整版本号管理文档（含 Android Studio 图形操作步骤） |

### 版本规则
- **versionName**: 取最近 git tag（去 v 前缀），有新 commit 则追加 +build.N.hash
- **versionCode**: Git 总 commit 数，保证单调递增
- 已打初始 tag **v1.0.0** 并推送到 Gitee

### 发版流程
1. 提交代码 → Push
2. Android Studio Git Log → 右键 commit → New Tag → 输入 v1.x.x → Push Tag
3. Build APK
4. 安装包自动显示新版本号
